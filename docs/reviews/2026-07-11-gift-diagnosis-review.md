## 3. Day5の実機確認で分かったこと

### 3.1 次へ・戻るが遠い

スマホで操作すると、「次へ」「戻る」を押すためにスクロールが必要になる場面があります。

改善方針：

- スクロールしなくても操作できる位置に置く
- 下部固定を検討する
- 固定する場合は本文や端末の安全領域へ重ねない
- 「戻る」は選択肢と違う見た目にする

### 3.2 検索キーワードをコピーしたい

診断結果の検索キーワードを、そのままECサイトへ貼り付けられる必要があります。

改善方針：

- 「検索ワードをコピー」ボタンを追加する
- コピー後に「コピーしました」と表示する
- スマホでも確実に動くようにする

### 3.3 検索キーワードが抽象的すぎる

「おしゃれ ギフト」程度では、候補を絞れません。

改善方針：

検索語を次の形にする。

> 予算 + 具体的な品名・ジャンル + 用途 + 相手との関係 + 安全条件

例：

> 3000円 個包装 焼き菓子 ギフト 職場 お礼

### 3.4 結果がジャンルで止まっている

おすすめジャンルを出すだけでは、ユーザーがさらに調べる必要があります。

結果ページに必要なもの：

- 第一候補
- 第二候補
- 避けた方がいいもの
- 避ける理由
- 迷ったときの無難な選び方
- 具体的な検索ワード
- 商品検索ボタン
- 渡すときの一言

### 3.5 情報が足りないなら設問追加も検討する

検索ワードの具体化に必要であれば、質問追加を検討します。

優先候補：

- 相手の年代

条件付き候補：

- 地域  
  体験ギフトや実店舗の候補を表示する場合だけ必要

ただし、質問数を増やす前に、既存回答から具体化できないか確認します。

### 3.6 体験ギフトには地域が必要

体験をおすすめする場合、地域が不明だと検索結果が広すぎます。

方針：

- 地域を質問する
- または、地域がない場合は体験ギフトを第一候補にしない
- 体験系だけ追加質問を出す方式も検討する

### 3.7 「次にやること」が弱い

診断後に何をすればいいかが十分に具体化されていません。

望ましい3ステップ：

1. 検索ワードをコピーする
2. Amazon・楽天・Yahooのいずれかで検索する
3. 第一候補に近い商品を3つだけ見比べる

結論

このサイトの一番の問題は、ターゲットの痛みが浅く扱われていることです。

ターゲットは、
「プレゼント選びが苦手」
「時間がかかる」
「苦行に感じる」
「優柔不断で決めきれない」
人ですよね。

でも、今のファーストビューは「相手・予算・目的に答えるだけで、外しにくいプレゼントの方向性がわかります」という説明で、かなり普通です。便利そうではあるけど、“あ、これ自分のことだ”という刺さり方が弱いです。

特に致命的なのは、サイト側が「商品を直接おすすめするのではなく、まずはプレゼントのジャンルを整理する診断」と明言している点です。
これは誠実ではありますが、ターゲットから見ると、結局まだ自分で探さないといけないの？ となります。

良いところ

5問だけで終わる設計は良いです。今のサイトは「5 questions」として、相手・目的・予算・相手の雰囲気・避けたいものを聞く形になっています。
プレゼント選びが苦手な人にとって、長い診断はそれだけで離脱理由になるので、軽さは武器です。

あと、「避けたいもの」を聞いているのも良いです。プレゼント選びで本当に怖いのは「喜ばれるか」よりも「外したらどうしよう」なので、NG条件を入れる方向性は正しいです。

辛口ポイント

1. 苦行感への共感が足りない

今の文章はきれいですが、ターゲットの本音に踏み込めていません。

たとえば、ターゲットはこう思っています。

「何をあげたらいいか分からない」
「センスないと思われたくない」
「調べ始めると候補が多すぎて決められない」
「結局Amazonや楽天を何十分も見て疲れる」
「無難すぎてもつまらないし、攻めすぎても怖い」

この感情に対して、今のサイトはまだ「便利ですよ」と言っているだけです。
もっと言うと、悩んでいる人の代弁ができていないです。

ファーストビューは、たとえばこういう方向の方が刺さります。

> プレゼント選び、毎回しんどくないですか？
相手・予算・避けたいものを選ぶだけで、外しにくい方向性を3分で整理します。



このくらい言った方が、ターゲットに届きます。

2. 診断結果が“決断”まで連れていっていない

結果画面には「おすすめジャンル」「選んだ理由」「検索キーワード」「次にやること」があります。
構成自体は悪くないです。

でも、プレゼント選びが苦手な人にとって、検索キーワードを渡されるだけではまだ重いです。

一番ほしいのは、ジャンルではなく、

「この条件なら、まずこの3方向から選べばいい」
「迷ったらこれ」
「これは避けた方がいい」
「検索するならこの言葉」
「渡す時の一言はこれ」

ここまでです。

つまり、現状は
“候補を整理するツール” であって、
“迷いを終わらせるツール” にはなっていません。

このターゲットに刺すなら、結果画面に必ず 最終判断の補助 が必要です。

例：

第一候補：消えものギフト

第二候補：実用雑貨

避けるべき：香りもの・サイズが関係するもの

迷ったら：お菓子＋小物のセット

検索ワード：3000円 おしゃれ お菓子 ギフト 個包装

添える一言：いつもありがとう。気軽に使ってもらえたら嬉しいです。


ここまで出すと、ユーザーはかなり楽になります。

3. 質問がまだ浅い

今の質問は、誰に贈るか、目的、予算、相手の雰囲気、避けたいものの5つです。
最低限としてはOKです。

ただ、プレゼント選びで重要な情報がまだ抜けています。

特に足りないのは、

相手の年代

相手との距離感

性別を考慮するかどうか

いつ渡すか

ネットで買うか、店舗で買うか

失敗したくない度

無難寄りか、少し印象に残したいか


このあたりです。

もちろん質問数を増やしすぎると重くなります。
なので、最初は5問でいいですが、結果画面で「もう少し絞り込む」ボタンを付けるのが良いです。

4. 競合と比べると“商品への近さ”が弱い

既存のギフト診断系を見ると、BIRTHDAY BARは「なにを贈ればいいかお悩みの方」に向けてギフト選びを手伝う構成で、シーン別に診断へ進ませています。

つまり競合は、診断 → 方向性 → 具体商品・ブランド まで近づけています。

一方で、このサイトは「ジャンル整理」で止まっています。
これは差別化にもなりますが、収益化やユーザー満足度の面では弱いです。

ターゲットはそこが苦手なので、最後の一歩まで背中を押す設計 にした方がいいです。

最終評価

今のサイトは、MVPとしては十分です。
ただ、公開して人に使ってもらうには、まだ弱いです。

特に弱いのはこの3つです。

1. トップでターゲットの悩みに刺さっていない


2. 診断結果が具体的な行動まで落ちていない


3. 収益化導線がまだ見えない



逆にここを直せば、かなり良くなります。

方向性としては、
「プレゼントを選んでくれるサイト」ではなく、
「プレゼント選びのストレスを減らして、候補を3つまで絞るサイト」
に寄せるのが一番強いです。
