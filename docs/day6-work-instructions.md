# プレゼント診断 Day6 作業指示書

- 作成日：2026-07-13
- 対象：<https://y-ai-lab.github.io/gift-diagnosis/>
- 推奨モデル：Luna
- 推論強度：中
- 状態：実装前・承認待ち

## 1. 今回の目的

診断結果ページの下部に、今後公開するnote記事とチェックリストの存在を伝える仮の案内を追加します。

まだ公開URLがないため、押せるリンクやボタンは作りません。「準備中」であることを明記し、押せない要素をリンクやボタンのように見せないことを優先します。

## 2. 現状理解

- GitHub Pagesで公開している静的サイトです。
- 結果ページは`index.html`に固定の画面構造があります。
- 診断結果の文言と候補は`script.js`で表示しています。
- Day6の案内は診断結果によって内容を変えないため、JavaScriptへ追加する必要はありません。
- 現在の第一候補、検索ワード、商品検索への導線を邪魔しない位置へ置きます。
- Day5完了ログに未コミットの更新があります。Day6ではこの既存変更を上書き・破棄しません。

## 3. 変更すること

結果ページの「今回の条件」の後、「もう一度診断する／回答を見直す」の前に、案内欄を1つ追加します。

案内欄の中には、次の2項目を置きます。

### 次に読むnote

- 表示ラベル：`準備中`
- 見出し：`プレゼント選びで迷ったときの3つの確認`
- 案内文：`診断結果を見たあと、商品を3つまで絞る手順を実例つきでまとめます。公開後、この場所から読めるようにします。`

### プレゼント選びチェックリスト

- 表示ラベル：`準備中`
- 見出し：`買う前に確認するチェックリスト`
- 案内文：`予算・渡しやすさ・相手が困らないかを確認できる、短いチェックリストを準備しています。`

## 4. 表示ルール

- 2項目とも押せない案内カードとして表示します。
- `a`要素や`button`要素は使用しません。
- 「準備中」を文字で表示し、色だけで状態を伝えません。
- 既存の結果カードと同じ角丸、枠線、余白を基本にします。
- 新しい強い色や大きな画像は追加しません。
- 375pxでは1列、広い画面では必要に応じて2列にします。
- 長い文章を詰め込まず、各項目を見出しと短い説明に分けます。

## 5. 想定変更ファイル

### `index.html`

- 結果ページ下部へDay6案内欄を追加します。
- `styles.css`のキャッシュ更新用クエリを1回更新します。

### `styles.css`

- 案内欄、準備中ラベル、2項目の配置に必要な最小限のスタイルを追加します。
- 既存の色、余白、角丸を再利用します。

### 変更しないファイル

- `script.js`
- `search-keywords.js`
- 質問・回答データ
- 診断結果データ
- Day5のnote・X原稿

## 6. 変更しないこと

- 質問数、質問文、選択肢、順番
- 配点、タイプ判定、同点処理
- 第一候補、第二候補、検索ワード
- Amazon・楽天・Yahooショッピングの検索リンク
- コピー機能
- 回答データの扱い
- 外部API、アクセス解析、データ保存
- 実在しないnote URLやチェックリストURL
- note記事やチェックリストの公開
- commit、push、GitHub Pages公開

## 7. 実装手順

1. 作業前のbranchと`git status`を再確認します。
2. `index.html`へ非リンクの案内欄を追加します。
3. `styles.css`へ必要最小限のスタイルを追加します。
4. 診断を最初から最後まで操作し、結果ページを表示します。
5. 375px、430px、デスクトップ幅で案内欄を確認します。
6. コンソール、ネットワーク、既存の検索・コピー機能を確認します。
7. 差分を自己レビューし、定型形式で完了報告します。

## 8. テスト

リポジトリに現在ある方法だけを使用します。

```text
node --check script.js
node --check search-keywords.js
node search-keywords.test.js
git diff --check
```

確認項目：

- トップから結果まで診断を完走できる
- 戻る、次へ、回答見直し、再診断が動く
- 第一候補、第二候補、検索ワードが従来どおり表示される
- 検索ワードをコピーできる
- 3モールの検索リンクが従来どおり開く
- Day6案内が結果ページだけに表示される
- 準備中の案内が押せる要素に見えない
- 新しい外部通信が増えていない
- コンソールエラーと404がない

## 9. 画面確認

### 375px

- 横スクロールなし
- 文字切れなし
- 2つの案内が1列で読みやすい
- 下部の再診断・回答見直しボタンと重ならない
- 「準備中」が見落とされない

### 430px

- 横スクロールなし
- 不自然な余白なし
- 見出しと説明文の折り返しが自然
- 押せない案内がボタンに見えない

### デスクトップ

- 内容が横へ広がりすぎない
- 既存カードと見た目がそろっている
- 2項目の情報量と高さが不自然に偏らない

## 10. 完了条件

- 仮のnote案内とチェックリスト案内が結果ページに表示される
- 両方に「準備中」と明記されている
- 実在しないリンクを追加していない
- 診断ロジックと結果データを変更していない
- 375px、430px、デスクトップで表示確認済み
- 既存テスト成功
- コンソールエラー、404、新しい外部通信なし
- Day5の未コミット変更を保持している
- commit、push、公開を行っていない

## 11. 停止条件

次の場合は推測で実装せず、作業を止めて確認を求めます。

- 実際のnote URLやチェックリストURLを追加する必要が出た
- 案内内容を診断結果ごとに変える必要が出た
- 診断ロジックや結果データの変更が必要になった
- Day5の未コミット変更と競合した
- 既存テストが失敗し、今回の変更が原因か判断できない
- push・公開が必要になった

## 12. 承認後の範囲

承認後は、`index.html`と`styles.css`の最小変更、テスト、3幅の画面確認、自己レビューまで行います。

commit、push、GitHub Pages公開は、別途明示的な指示があるまで行いません。
